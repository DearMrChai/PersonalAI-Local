# PersonalAI-Local 本地AI整合平台

## 项目目标

基于Vue3 + Node.js + 本地大模型（llama.cpp）+ ComfyUI，搭建**轻量、可扩展、私有化**的AI整合平台，实现「角色聊天+AI绘图」一体化，支持自定义角色、动态绘图工作流、服务状态检测，最终打包为绿色EXE文件，双击即用，无需额外环境。

### 核心诉求

1. 前后端分离，架构清晰，易扩展（后续支持群聊、世界观、多模型切换）；
2. 所有外部路径（llama.cpp/ComfyUI/模型）可配置，项目为「空包」，易分享；
3. 实时检测服务状态，避免启动失败无感知；
4. 绘图工作流可配置，替代硬编码，支持多风格切换；
5. 最终打包为单EXE，绿色免安装，他人可直接使用。

## 项目架构

**前后端彻底分离**，前端负责界面交互，后端负责接口逻辑/进程管理/AI调用，所有配置可持久化，外部依赖可配置。

### 技术栈
| 模块       | 技术选型                  | 核心作用                     |
|------------|---------------------------|------------------------------|
| 前端       | Vue3 + Vite + Element Plus | 界面交互、路由管理、状态展示 |
| 前端路由   | Vue Router 4              | 配置页/主布局切换            |
| 后端       | Node.js + Express         | 接口提供、跨域处理           |
| 实时通信   | WebSocket（ws）           | 聊天流式输出、可打断         |
| 进程管理   | child_process             | 启动/关闭llama.cpp/ComfyUI   |
| AI绘图     | ComfyUI HTTP API          | 绘图任务提交/状态查询        |
| 本地大模型 | llama.cpp HTTP/WS API     | 流式聊天、自定义参数         |
| 打包工具   | pkg                       | 后端+前端静态文件打包为EXE   |
| 数据持久化 | 本地JSON文件              | 配置/角色数据存储（预留数据库） |

## 快速开始（开发环境）
### 前置条件
1. 安装Node.js（建议v18+，与后端pkg打包targets一致，执行`node -v`查看版本）；
2. 已下载llama.cpp（CPU版推荐，通用性最强）、GGUF模型、ComfyUI；
3. 前端/后端均已安装依赖（执行对应目录下的`npm install`）。

### 运行步骤
1. 打开项目根目录，双击`start-all.bat`，一键启动前端（http://localhost:5173）+ 后端（http://localhost:3000）；
2. 浏览器访问`http://localhost:5173`，进入**初始化配置页**；
3. 填写核心配置（必选）：
   - llama-server.exe 绝对路径；
   - GGUF模型 绝对路径；
   - ComfyUI启动脚本（start.bat）绝对路径；
4. 可选配置：调整llama端口/上下文长度/GPU分层、ComfyUI端口；
5. 点击「一键启动 Llama + ComfyUI」，等待服务启动（前端显示绿灯即成功）；
6. 点击「保存并进入系统」，跳转到主布局，即可进行「角色管理+聊天+绘图」操作。

## 打包为EXE（绿色免安装）
### 前置条件
1. 开发环境可正常运行；
2. 首次打包需安装打包依赖（仅执行一次）。

### 打包步骤
1. 项目根目录双击`01-安装打包依赖.bat`，自动安装pkg打包工具及后端依赖；
2. 依赖安装完成后，双击`02-一键打包成EXE.bat`，脚本自动执行：
   - 前端打包：生成`frontend/dist`静态文件；
   - 后端打包：将Node环境+后端代码+前端静态文件打包为EXE；
   - 输出目录：项目根目录`release/`下生成`PersonalAI-Local.exe`；
3. 打包完成后，直接双击`release/PersonalAI-Local.exe`，启动服务后访问`http://localhost:3000`即可使用。

### EXE使用说明
1. EXE为**绿色单文件**，无需安装Node/Vue/任何环境，双击即用；
2. 首次使用需在配置页填写llama/ComfyUI/模型路径，配置后自动持久化；
3. 服务启动后，黑窗可最小化，不可关闭（关闭则服务停止）；
4. 如需关闭服务，可直接关闭黑窗，或后续添加「一键关闭脚本」。

## 常见问题
### 1. 启动服务后，前端显示红灯（服务未启动）？
- 检查配置页填写的路径是否为**绝对路径**，且文件存在；
- 检查端口是否被占用（llama默认5001，ComfyUI默认8188），可在配置页修改端口；
- 查看后端黑窗日志，是否有报错信息（如文件不存在、权限不足）；
- 若为llama.cpp CUDA版，检查显卡驱动/CUDA环境是否安装，建议先使用CPU版测试。

### 2. 打包EXE失败，提示「文件不存在」？
- 检查`backend/package.json`的`pkg.assets`配置，是否包含所有静态文件（config/workflows/frontend/dist）；
- 确保前端已打包生成`frontend/dist`文件夹，打包脚本会自动执行`npm run build`，若失败需手动在前端目录执行；
- 检查项目路径是否包含**中文/空格**，pkg打包不支持中文路径，建议项目放在纯英文路径下（如D:/PersonalAI-Local/）。

### 3. 绘图生成成功，但未显示图片？
- 检查ComfyUI是否正常启动（访问ComfyUI地址http://localhost:8188可打开界面）；
- 检查ComfyUI的`output`目录是否有生成的图片文件；
- 前端图片链接为ComfyUI的view接口，确保端口与配置页一致。

### 4. 聊天无响应，WebSocket连接失败？
- 检查后端是否正常启动（访问http://localhost:3000可打开前端页面）；
- 检查浏览器控制台是否有跨域报错（后端已配置cors，无需额外处理）；
- 检查llama.cpp是否正常启动，且端口与配置页一致（默认5001）。

## 项目约定
1. **代码规范**：前端遵循Vue3组合式API规范，后端遵循ES6模块规范，变量/函数命名使用小驼峰；
2. **配置规范**：所有可配置项均在`backend/config/default.config.js`中定义，用户配置自动覆盖默认配置，不修改默认配置文件；
3. **路径规范**：所有外部依赖路径均为绝对路径，由用户在配置页选择，代码中不写死任何本地路径；
4. **扩展规范**：新增功能按「后端加路由+服务 → 前端加页面/组件 → 路由配置」的流程，不修改原有代码，实现无侵入扩展；
5. **打包规范**：不将大模型/ComfyUI/llama.cpp本体打进EXE，仅打包项目代码和Node运行环境，保持EXE轻量。
